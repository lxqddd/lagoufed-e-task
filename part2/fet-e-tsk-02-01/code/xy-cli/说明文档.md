#### 使用Nodejs自定义脚手架工具说明文档

1. 创建一个空文件夹`xy-cli`
2. 执行`npm init -y`初始化`package.json`文件
3. 在该目录下创建一个名为`cli.js`的文件，作为脚手架的控制文件
   1. 在该文件的头部要添加一行代码`\#! /usr/bin/env node`
   2. 他是用来解决不同的用户node路径不同的问题，可以让系统动态的去查找node来执行你的脚本文件。
4. 创建一个`templates`目录，用来存放需要生成的模板文件

```javascript
#! /usr/bin/env node
const fs = require('fs')
const path = require('path')
const inquirer = require('inquirer')
const ejs = require('ejs')

inquirer
  .prompt([	// 在命令行窗口中向用户询问一些问题，用于对模板中相关位置定义的变量进行替换
    {
      type: 'input',
      name: 'name',
      message: 'Project name?'
    }
  ])
  .then(anwsers => {
	// anwsers 是收集到的用户的答案

    //根据目录
    const tmplDir = path.join(__dirname, 'templates')
    //目标目录
    const destDir = process.cwd()

    //将模板下的文件全部转换到目标目录
    fs.readdir(tmplDir, (err, files) => {
      if (err) throw err
      files.forEach(file => {
        // 通过模板引擎渲染文件
        ejs.renderFile(path.join(tmplDir, file), anwsers, (err, result) => {
          if (err) throw err

          // 将结果写入目标文件路径
          fs.writeFileSync(path.join(destDir, file), result)
        })
      })
    })
  })

```

5. 执行`npm link`将该脚手架工具`link`到全局
6. 创建要给名为test的目录, 在该目录下执行`xy-cli`，就会生成对应的模板文件
